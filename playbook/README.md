# Playbook

Patterns, anti-patterns, conventions, and guardrails for building in this repo. Agents read these before executing plans. Humans update them during PR review.

The playbook starts empty and grows from real work. Every time you review a PR and think "the agent shouldn't have done that," add a rule here. Every time the agent does something well that should be repeated, capture the pattern.

Organize by topic: `testing.md`, `data-layer.md`, `frontend.md`, whatever fits the project. Start with a single `conventions.md` if you're not sure how to split things up.

See the Playbook section in [CDD.md](../CDD.md) for more on how this works.
